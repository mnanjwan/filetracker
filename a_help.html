Guide to using FileTrakA!

  • Click the Log In button on the landing page.
  • Enter your service number with the prefix "NCS".
  • Enter your password.
  • Click the Login button or press Enter.

  • Click on Change Password either on the landing page or on the login page
  • On the login page, click Forgot Password? under the login button. This will take you to the Forgot Password page.
  • Perform the following steps
    • o Enter your service number.
      o Enter your email address and click Send One-Time Password (OTP).
      o You will receive a 7-digit OTP.
      o Enter the OTP and change your password.

    1. Click the Create New File icon on the left side of the dashboard. This will open the Create File Form.
    2. Select Create File under the Action dropdown.
    3. In the File Number field, enter the file number. (The file number will have a predictive prefix of "NCS-", so do not add a backslash as the system will interpret it as a path.)
    4. Select the precedence marking from the dropdown.
    5. Type the File Description.
    6. Click Create File or press Enter.

    1. Click Dispatch Files. The Dispatch File dialog box will appear.
    2. You can search for a file either by File ID or File Description.
    3. Once you provide the necessary information, the auto-search boxes will display a list of options.
    4. Select the appropriate Command, Unit, and Precedence Marking.
    5. Click the Dispatch button to send the file.

    1. To track a file and view the audit trail, enter the File ID.
    2. If the File ID is not available, you can search using the File Description.
    3. If the file is within your purview, it will appear along with its details.
    4. Copy the File ID to conduct a proper audit trail.

    1. Log in to your dashboard. All files sent to you will appear in order from newest to oldest.
    2. Ensure you have received the file, then click Receive File.
    3. A prompt will appear asking if you are sure you have received the file. Click OK.
    4. The file will automatically move from Files Yet to be Received by Unit to Files Received by Unit.
